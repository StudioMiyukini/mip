## 4. Qualite des annotations MSCM (echantillon 20 fichiers)

### 4.1 IDs dupliques

**38 identifiants @id dupliques** trouves dans le workspace. Echantillon :

- `auth_can_access_edition` — present dans plusieurs crates service
- `auth_sign_in`, `auth_sign_out`, `auth_sign_up` — IDs trop generiques, conflits inter-services
- `jayxpose_auth_sign_in`, `jayxpose_auth_sign_out`, `jayxpose_auth_sign_up` — doublon JayXpose
- `borderguard_boundary_type` — doublon dans borderguard
- `kindmother_sync_delta_operation` — doublon KindMother
- `miyubooking_create_booking` — doublon MiyuBooking

**Verdict** : DEFAUT. Les IDs doivent etre **globalement uniques**. Les doublons proviennent principalement d'annotations sur des fonctions similaires dans des modules differents sans prefixe discriminant.

**Verdict** : DEFAUT MAJEUR. Seulement 169 annotations sur ~1782 utilisent un chiffre conforme (9.5%). Deux conventions coexistent : noms de strate en texte (ancienne) vs chiffres 0-9 (convention CLAUDE.md). Il faut harmoniser.

### 4.3 Annotations incompletes

**30+ fichiers** annotes avec @id mais manquant un ou plusieurs des champs obligatoires (@do, @role, @layer, @human). Les patterns incomplets :

### 4.5 Formats heterogenes

Les deux formats sont valides mais l'heterogeneite complique les outils d'indexation.

## 8. Recommandations

### Priorite 1 — Harmoniser @layer (F-02)

Choisir une convention unique et migrer. Deux options :
- **Option A** : Forcer les chiffres 0-9/K comme CLAUDE.md l'exige. Migration lourde (~1600 annotations).
- **Option B** : Accepter les noms de strate comme alias valides et documenter le mapping (core=4, tool=6, operator=7, kernel=K, etc.). Migration legere.

Recommandation Arianne : **Option B** avec un script de validation qui accepte les deux formats.

### Priorite 2 — Annoter les apps (F-01)

Ordre de priorite :
1. `apps/central/` (170 fichiers, 17% -> objectif 50%)
2. `apps/miou-llm-bridge/` (21 fichiers, 0% -> service critique)
3. `apps/jayfestival/` (40 fichiers, 0% -> plus gros service standalone)
4. `apps/origin/` (31 fichiers, 6% -> serveur expose)

### Priorite 3 — Deduplication @id (F-03)

Prefixer systematiquement les IDs avec le nom du crate/module pour eviter les collisions.
Ex: `auth_sign_in` -> `jayfestival_auth_sign_in`, `jayxpose_auth_sign_in`.

### Priorite 4 — Completer les annotations partielles (F-04)

Script automatise pour detecter les fichiers avec @id mais sans @human ou @role, puis completer.
