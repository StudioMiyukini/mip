# P2-L3d-D4-01 + P2-L3d-D4-02 -- Audit UI/UX miyuki-ui-dioxus

**Date** : 2026-03-02
**Auditeur** : Lise (Dev Front-End, Direction Artistique)
**Referentiels** : WCAG 2.2 (W3C), ISO 9241 (Parts 11, 110, 210)
**Perimetre** : `crates/miyuki-ui-tokens/` (15 fichiers), `crates/miyuki-ui-dioxus/` (47 fichiers)
**Cible conformite** : WCAG AA, ISO 9241-110 (7 principes)

---

## TL;DR

Score global **62/100**. La librairie miyuki-ui presente une architecture solide (Atomic Design, design tokens, themes const) et de bonnes pratiques ARIA ponctuelles. Cependant, 9 defauts WCAG AA bloquants sont identifies : absence de focus visible sur Button/Input, contraste insuffisant blanc-sur-accent (2.82:1), absence de keyboard navigation, et pas de focus trap dans Modal. Cote ISO 9241, la controllabilite (undo/cancel) et la tolerance aux erreurs sont partiellement couvertes. Le D2 theme a un body text a 12px qui est acceptable mais les tailles xs (8px) et sm (10px) sont trop petites pour un usage corps de texte.

## 5. Plan de correction recommande

### Sprint 1 -- Focus et clavier (CRITIQUE, 8 composants)

**Priorite maximale.** La navigation clavier est le defaut le plus repandu.

1. **Ajouter un style focus ring global** dans `styles.rs` :
   ```rust
   pub fn focus_ring(color: &Rgba) -> String {
       format!("outline: 2px solid {}; outline-offset: 2px;", color.to_css())
   }
   ```
2. **Button** : Remplacer `outline: none` par un `:focus-visible` style (ou listener onfocus dans Dioxus)
3. **TextInput** : Idem, ajouter focus ring sur border-color change
4. **Checkbox** : Ajouter `tabindex: "0"`, handler `onkeydown` (Space = toggle)
5. **Toggle** : Idem Checkbox
6. **Tooltip** : Ajouter `onfocus`/`onblur` en plus de mouseenter/mouseleave
7. **Dropdown** : Ajouter `role="menu"`, items `role="menuitem"`, navigation fleches, Escape close
8. **TabItem/TabBar** : Ajouter `tabindex`, navigation fleches, `role="tab"` deja present
9. **Breadcrumb** : Transformer les span clickables en `<button>` ou ajouter `tabindex`
10. **MenuItem** : Ajouter `tabindex`

### Sprint 2 -- Contraste tokens (CRITIQUE, 3 tokens)

1. **T-01** : Assombrir `accent_primary` de #1a9fff vers ~#0d6ebd (ratio >= 4.5:1 avec blanc) **OU** utiliser un texte plus fonce (ex: bg_base) sur les boutons primaires
2. **T-03** : Eclaircir `error` de #e04040 vers ~#ef5350 ou utiliser `text_high` (#fff) seulement sur fond error (pas l'inverse)
3. **T-06** : D2 xs de 8px vers 10px minimum

### Sprint 3 -- Modal et overlays (CRITIQUE)

1. **Modal** : Focus trap (pieger Tab/Shift+Tab dans le modal), Escape handler, auto-focus premier interactif, restauration focus a la fermeture
2. **CommandPalette** : Idem + navigation fleches dans les items
3. **Skip navigation** : Ajouter dans DashboardLayout et ServiceLayout

### Sprint 4 -- Association label/input et ARIA (MAJEUR)

1. **TextInput** : Prop `id` avec generation auto si absent, `aria-describedby` vers le message d'erreur
2. **FormField** : `<label for={id}>` lie au child input
3. **DataTable** : `scope="col"` sur `<th>`, prop `caption` optionnelle, `aria-sort`
4. **Form** : Utiliser `<form>` au lieu de `<div>`, ou ajouter `role="form"`
5. **SearchBar** : Ajouter `aria-label="Search"` sur l'input
6. **Text** : Utiliser des elements natifs (h1-h6, p, code) via blocs conditionnels

### Sprint 5 -- ISO 9241 ameliorations (RECOMMANDE)

1. **ConfirmDialog** : Nouveau organism pour les actions destructives
2. **Button loading** : Integrer Spinner + `aria-busy="true"`
3. **ProgressBar** : Prop `label` pour `aria-label`
