### Observations generales

- **Zero `unsafe`** : Conforme.
- **Zero `panic!`** : Conforme.
- **1 seul `unwrap()` en production** (unc_auth.rs:257) : A corriger.
- **Gestion d'erreurs DB** : Pattern recurrent de `let _ = db.xxx()` qui ignore silencieusement les echecs des operations d'ecriture. Les lectures utilisent correctement `unwrap_or_default()`.
- **Duplication** : Le pattern "charger participations + map exposants" est repete 4 fois dans org_exposants.rs et org_plan.rs.
- **MSCM** : 0/20 fichiers annotes. Deficit significatif par rapport a l'objectif de 50%.
- **Tests** : Aucun fichier de ce perimetre ne contient de tests unitaires. Attendu pour des composants UI Dioxus (les tests d'integration sont dans le crate `jayfestival`).
- **Securite auth** : Le flux d'authentification (unc_auth.rs) est fonctionnel mais minimaliste. Mot de passe faible accepte (6 chars), pas de rate limiting visible, pas de CSRF token.
