## Findings

| # | Sev | WCAG/ISO Ref | Fichier:Ligne | Probleme | Impact | Fix |
|---|-----|-------------|---------------|----------|--------|-----|
| 1 | CRITIQUE | WCAG 2.1.1 (A) Keyboard | Tous fichiers | Aucun `tabindex` sur les elements interactifs non-natifs. Les `div` avec `onclick` (cards, items) ne sont pas navigables au clavier. | Utilisateurs clavier/lecteur ecran bloques, navigation impossible sans souris. | Ajouter `tabindex: "0"`, `onkeypress` (Enter/Space) sur tout `div`/`span` cliquable, ou migrer vers `button`. |
| 2 | CRITIQUE | WCAG 2.4.7 (AA) Focus Visible | Tous fichiers | Aucune gestion du focus visible. Les `button` ont `border: none` sans `:focus-visible` ou outline alternatif. Dioxus inline styles ne supportent pas les pseudo-classes. | Focus invisible, utilisateurs clavier perdus dans l'interface. | Implementer un systeme de focus via signal `is_focused` + `onfocus`/`onblur` handlers avec style conditionnel (ring bleu 2px). |
| 3 | CRITIQUE | WCAG 4.1.2 (A) Name/Role/Value | Tous fichiers | Zero attribut ARIA dans toute l'app : pas de `aria-label`, `aria-role`, `aria-pressed`, `aria-expanded`, `role="alert"`, `aria-live`. | Technologies assistives ne peuvent pas interpreter l'interface. | Ajouter systematiquement les ARIA roles/labels sur tous les widgets interactifs. |
| 4 | CRITIQUE | WCAG 2.4.1 (A) Bypass Blocks | mod.rs:~200 | Pas de mecanisme "skip to content". La sidebar + header sont traverses entierement avant d'atteindre le contenu principal. | Navigation fastidieuse au clavier (30+ elements avant le contenu). | Ajouter un lien "skip to main content" en premier element avec ancre vers la zone principale. |
| 5 | MAJEUR | WCAG 1.3.1 (A) Info/Relations | unc_auth.rs:~120-180 | Les `label` ne sont pas lies programmatiquement aux `input` : pas d'attribut `for`/`id`. Pattern identique dans tous les formulaires (org_parametres, org_editions, org_compte, exp_compte, vis_compte). | Lecteurs ecran ne peuvent pas associer labels aux champs. Erreurs de saisie probables. | Ajouter `id` unique sur chaque `input` et `r#for: "{id}"` sur le `label` correspondant. |
| 6 | MAJEUR | WCAG 1.4.3 (AA) Contrast Min | Tous fichiers | `text_muted` (#5c6873) sur `bg_secondary` (#232f3e) = ratio ~2.3:1 (requis 4.5:1). Utilise massivement pour dates, descriptions, hints. | Texte illisible pour les utilisateurs malvoyants ou en environnement eclaire. | Augmenter `text_muted` a minimum #8a9199 (ratio 4.5:1 sur #232f3e) ou utiliser `text_secondary`. |
| 7 | MAJEUR | WCAG 1.4.3 (AA) Contrast Min | Tous fichiers | `text_secondary` (#8f98a0) sur `bg_secondary` (#232f3e) = ratio ~4.2:1 (requis 4.5:1 pour texte <18px). | Legere sous-conformite pour le texte normal. | Eclaircir `text_secondary` a #99a2aa minimum, ou utiliser font-size >= 18px/bold >= 14px pour ces cas. |
| 8 | MAJEUR | WCAG 1.1.1 (A) Non-text Content | Tous fichiers | Emojis utilises comme icones semantiques sans alternative textuelle : "🎪" (evenement), "🏢" (organisateur), "🧑‍💼" (exposant), "📝", "💰", "🔧", etc. Au moins 50+ occurrences. | Lecteurs ecran vocalisent les noms Unicode des emojis (ex: "tent") au lieu de la signification contextuelle. | Encapsuler chaque emoji dans `span { aria_hidden: "true" }` et ajouter un `span` sr-only avec le texte alternatif, ou utiliser `aria_label` sur le conteneur parent. |
| 9 | MAJEUR | WCAG 3.3.1 (A) Error Identification | org_editions.rs:~45-60 | La validation du formulaire de creation d'edition ne verifie que si le nom est vide, sans message d'erreur accessible (pas de `role="alert"`, pas de `aria-describedby`). | Utilisateurs ne comprennent pas pourquoi la soumission echoue. | Ajouter un signal `error_msg`, afficher dans un `div { role: "alert" }` avec texte descriptif. |
| 10 | MAJEUR | WCAG 3.3.1 (A) Error Identification | exp_compte.rs:82-91, vis_compte.rs:59-66 | Bouton "Enregistrer" sans feedback de succes/erreur apres action. Pattern identique dans org_parametres.rs et exp_fiche_publique.rs. | Utilisateur ne sait pas si la sauvegarde a reussi. | Ajouter un signal `save_status`, afficher une notification avec `aria-live: "polite"`. |
| 11 | MAJEUR | ISO 9241-110 7.3 Conformity Expect. | org_documents.rs:150-152 | Bouton "Supprimer" sur document sans boite de confirmation. Action destructive immediate. | Perte de donnees accidentelle par clic errone. | Ajouter une modale de confirmation : "Etes-vous sur de vouloir supprimer ce document ?". |
| 12 | MAJEUR | ISO 9241-110 7.3 Conformity Expect. | org_compte.rs:~241 | Bouton "Supprimer le compte" sans confirmation. Action irreversible en un clic. | Suppression accidentelle du compte organisateur et toutes ses donnees. | Modale de confirmation + saisie du nom du compte pour valider la suppression. |
| 13 | MAJEUR | ISO 9241-110 7.3 Conformity Expect. | exp_compte.rs:194-197 | Bouton "Supprimer" le compte exposant sans confirmation. | Suppression accidentelle du compte exposant. | Modale de confirmation avec double validation. |
| 14 | MAJEUR | ISO 9241-110 7.3 Conformity Expect. | vis_compte.rs:204-206 | Bouton "Supprimer" le compte visiteur sans confirmation. | Suppression accidentelle + perte des points/billets/reservations. | Modale de confirmation avec mention des donnees perdues. |
| 15 | MAJEUR | ISO 9241-110 7.3 Conformity Expect. | org_equipe.rs:~163 | Bouton "Retirer" un membre d'equipe sans confirmation. | Retrait accidentel d'un collaborateur. | Modale de confirmation avec nom du membre. |
| 16 | MAJEUR | ISO 9241-110 7.3 Conformity Expect. | org_exposants.rs:~186-189 | Bouton "Refuser" une candidature exposant sans confirmation. | Refus accidentel irreversible d'un exposant. | Modale de confirmation + champ motif de refus optionnel. |
| 17 | MAJEUR | ISO 9241-110 7.3 Conformity Expect. | org_programme.rs:~248-252 | Bouton de suppression d'animation sans confirmation. | Perte de donnees du programme evenementiel. | Modale de confirmation. |
| 18 | MAJEUR | WCAG 4.1.2 (A) Name/Role/Value | org_services.rs:138-143 | Toggle switches custom (div cliquable) sans `role="switch"`, sans `aria-checked`, sans gestion clavier. Pattern identique dans exp_fiche_publique.rs:249-255, exp_notifications.rs:258-276, vis_compte.rs:255-260. | Impossible d'utiliser les toggles sans souris. Etat invisible pour les technologies assistives. | Ajouter `role: "switch"`, `aria_checked: "{state}"`, `tabindex: "0"`, `onkeypress` (Space/Enter). |
| 19 | MAJEUR | WCAG 1.4.1 (A) Use of Color | Tous fichiers (Badge, status) | Les badges de statut (publie/brouillon/acceptee/refusee) communiquent le statut uniquement par couleur + texte. La couleur est redondante avec le texte (bon). Mais les `PointsHistoryRow` (vis_compte.rs:269-273) utilisent vert/rouge sans symbole +/- visible pour les daltoniens. | Information de sens (positif/negatif) potentiellement perdue pour les daltoniens. | Ajouter un indicateur non-colorimetrique : fleche haut/bas, ou prefixe textuel explicite sur le montant. |
| 20 | MAJEUR | WCAG 3.3.2 (A) Labels/Instructions | unc_landing.rs:~64-72 | Barre de recherche sans `aria-label` ni `label` visible associe. Seul un `placeholder` existe. | Lecteurs ecran n'annoncent pas la fonction du champ. | Ajouter `aria_label: "Rechercher un evenement, organisateur ou exposant"` sur l'input. |
| 21 | MINEUR | ISO 9241-110 7.2 Self-descriptiveness | Tous fichiers | Aucun indicateur de chargement pour les operations DB synchrones (recherche, listes, filtres). L'UI reste figee pendant les requetes. | Incertitude utilisateur : "l'app a-t-elle plante ?" | Ajouter un spinner ou skeleton loader pendant les operations DB. |
| 22 | MINEUR | WCAG 1.3.1 (A) Info/Relations | Tous fichiers | Pas de landmarks ARIA (`role="navigation"`, `role="main"`, `role="complementary"`) sur sidebar, contenu principal, sections. | Structure semantique invisible pour les technologies assistives. | Ajouter `role: "navigation"` sur la sidebar, `role: "main"` sur le contenu, `role: "complementary"` sur les panneaux secondaires. |
| 23 | MINEUR | WCAG 2.4.6 (AA) Headings/Labels | org_annonces.rs, org_budget.rs | Certaines pages utilisent des donnees de demo statiques sans indication claire a l'utilisateur que ces donnees ne sont pas reelles. | Confusion possible entre vraies donnees et donnees de demonstration. | Ajouter un bandeau "Donnees de demonstration" visible en haut de la page quand les donnees sont statiques. |
| 24 | MINEUR | ISO 9241-110 7.5 Error Tolerance | vis_reservations.rs:176-177 | Bouton "Annuler" une reservation sans confirmation ni possibilite d'annulation de l'annulation. | Annulation accidentelle d'une reservation payee. | Ajouter confirmation + mention de la politique d'annulation. |
